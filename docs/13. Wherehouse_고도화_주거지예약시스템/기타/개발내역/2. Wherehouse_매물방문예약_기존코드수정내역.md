### 2.1 `JWT/SecurityConfig/SecurityConfig.java`

**경로**: `wherehouse/src/main/java/com/wherehouse/JWT/SecurityConfig/SecurityConfig.java`

`reviewServiceFilterChain` 직후, 클래스 닫는 `}` 직전에 **두 개의 신규 `@Bean` 메서드**가 추가되었다. 기존 7 개 필터 체인 Bean 의 코드는 한 줄도 변경하지 않았다.

#### 추가 (1) — `visitReservationPageFilterChain`
회원 페이지 4 종 (`/visit/me/**`) 인증 처리.

```java
@Bean
public SecurityFilterChain visitReservationPageFilterChain(HttpSecurity http) throws Exception {
    http.securityMatcher("/visit/me/**")
            .authorizeHttpRequests(auth -> auth.anyRequest().authenticated())
            .csrf(csrf -> csrf.disable())
            .sessionManagement(sess -> sess.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .addFilterAt(new JwtAuthProcessorFilter(cookieUtil, jwtUtil, env), UsernamePasswordAuthenticationFilter.class)
            .exceptionHandling(exception ->
                    exception.authenticationEntryPoint(new JwtAuthenticationFailureHandler())
                            .accessDeniedHandler(new JwtAccessDeniedHandler())
            );
    return http.build();
}
```

#### 추가 (2) — `visitReservationServiceFilterChain`
방문 예약 REST API 전 경로 (`/api/v1/visit/**`) 인증·인가.

```java
@Bean
public SecurityFilterChain visitReservationServiceFilterChain(HttpSecurity http) throws Exception {
    http.securityMatcher("/api/v1/visit/**")
            .authorizeHttpRequests(auth -> auth
                    .requestMatchers(HttpMethod.GET, "/api/v1/visit/properties/*/slots").permitAll()  // F003 선택적 인증
                    .requestMatchers(HttpMethod.POST,   "/api/v1/visit/**").authenticated()
                    .requestMatchers(HttpMethod.DELETE, "/api/v1/visit/**").authenticated()
                    .requestMatchers(HttpMethod.PATCH,  "/api/v1/visit/**").authenticated()
                    .requestMatchers(HttpMethod.GET,    "/api/v1/visit/**").authenticated()
                    .anyRequest().denyAll()
            )
            .csrf(csrf -> csrf.disable())
            .sessionManagement(sess -> sess.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .addFilterAt(new JwtAuthProcessorFilter(cookieUtil, jwtUtil, env), UsernamePasswordAuthenticationFilter.class)
            .exceptionHandling(exception ->
                    exception.authenticationEntryPoint(new ApiAuthenticationEntryPoint())
                            .accessDeniedHandler(new JwtAccessDeniedHandler())
            );
    return http.build();
}
```

**근거**: 설계 명세서 3.1 인증 정책 표 — `GET /api/v1/visit/properties/{propertyId}/slots` 만 비인증 허용, 그 외 모든 메서드·경로는 인증 필수.

**경로**: `wherehouse/src/main/java/com/wherehouse/page/PageResponseController.java`

#### 변경 (1) — import 추가

#### 변경 (2) — 회원 페이지 4 종 GET 라우팅 추가
클래스 끝부분에 4 개 메서드 + 헬퍼 1 개 추가:

```java
// ========================================================================
// 방문 예약 회원 페이지 4종 (설계 명세서 섹션 6.8, 7.9~7.13)
// SecurityConfig.visitReservationPageFilterChain 이 /visit/me/** 에 JWT 인증을 강제하므로
// 본 컨트롤러는 @AuthenticationPrincipal 만으로 안전하게 currentUserId 를 추출한다.
// ========================================================================

@GetMapping("/visit/me/reservations")
public String visitMeReservations(@AuthenticationPrincipal String userId, Model model) {
    model.addAttribute("currentUserId", normalizeUserId(userId));
    return "visit/me_reservations";
}

@GetMapping("/visit/me/subscriptions")
public String visitMeSubscriptions(@AuthenticationPrincipal String userId, Model model) {
    model.addAttribute("currentUserId", normalizeUserId(userId));
    return "visit/me_subscriptions";
}

@GetMapping("/visit/me/slots")
public String visitMeSlots(@AuthenticationPrincipal String userId, Model model) {
    model.addAttribute("currentUserId", normalizeUserId(userId));
    return "visit/me_slots";
}

@GetMapping("/visit/me/notifications")
public String visitMeNotifications(@AuthenticationPrincipal String userId, Model model) {
    model.addAttribute("currentUserId", normalizeUserId(userId));
    return "visit/me_notifications";
}

/** AnonymousAuthenticationFilter 가 주입하는 "anonymousUser" 를 null 로 정규화. */
private String normalizeUserId(String userId) {
    if (userId == null || "anonymousUser".equals(userId)) return null;
    return userId;
}
```

**근거**: UI 작업 #4. 회원 페이지 4 종이 SecurityConfig 의 신규 필터 체인을 통과해 `userId` 를 SecurityContext 에 보유한 상태로 도달하므로, `@AuthenticationPrincipal` 만으로 안전 추출.

### 3.3 `static/js/property_board.js`

`buildCard(p)` 함수 내 버튼 영역 한 블록을 다음과 같이 교체.

```javascript
// (변경 전)
var actionsHtml = '';
if (currentUserId && p.status === 'ACTIVE') {
    var canEdit = true; // 인증된 사용자 누구나 수정 가능
    var canChangeStatus = (p.registered_user_id && currentUserId === p.registered_user_id);

    actionsHtml = '<div class="pb_card_actions" onclick="event.stopPropagation();">';

// (변경 후)

    if (!isOwner) {
        var aptNmEscaped = (p.apt_nm || '').replace(/'/g, "\\'");
        actionsHtml += '<button class="pb_card_btn pb_card_btn_reserve" onclick="window.openVisitSlotPicker(\'' +

    if (currentUserId && isOwner) {
        // 방문 예약 도입에 따른 정책 강화: 수정·상태 변경 모두 등록자 본인만 가능.
        actionsHtml += '<button class="pb_card_btn pb_card_btn_edit" onclick="openEditModal(...)">수정</button>';

        actionsHtml += '<button class="pb_card_btn pb_card_btn_delete" onclick="openStatusModal(..., \'DELETED\')">삭제</button>';

    }
    actionsHtml += '</div>';
```

```javascript
// (변경 전)
let buttonsHtml = '';
if (property.can_edit || property.can_change_status) {
    buttonsHtml = '<div class="property_card_actions">';
    if (property.can_edit) {
        buttonsHtml += `<button class="property_action_btn edit" onclick="openEditModal(...)">수정</button>`;
    }
    if (property.can_change_status) {
        buttonsHtml += `<button class="property_action_btn complete" onclick="changePropertyStatus(..., 'COMPLETED')">거래완료</button>`;
        buttonsHtml += `<button class="property_action_btn delete" onclick="changePropertyStatus(..., 'DELETED')">삭제</button>`;
    }
    buttonsHtml += '</div>';
}

// (변경 후)
// 방문 예약 기능: 등록자 본인이 아닌 경우(또는 비로그인) "방문 예약" 버튼을 항상 노출.
//                ACTIVE 상태인 매물에 한해 노출하며, 자기 매물 예약 불가는 서버가 거부한다.
let buttonsHtml = '<div class="property_card_actions">';
const propertyNameForBtn = (property.property_name || '').replace(/'/g, "\\'");
if (!property.owned_by_current_user && property.status !== 'COMPLETED' && property.status !== 'DELETED') {
    buttonsHtml += `<button class="property_action_btn reserve" onclick="window.openVisitSlotPicker('${property.property_id}', 'CHARTER', '${propertyNameForBtn}')">방문 예약</button>`;
```

## 4. 변경의 정책적 의미 요약

본 1 차에서 가해진 정책 변경은 단 하나의 일관된 명제로 요약된다:

> **매물 자원의 책임 주체 (수정·상태 변경) 와 그 매물에 묶인 방문 예약 자원의 책임 주체 (윈도우 공개·철회·결과 분류) 는 모두 매물 등록자(REGISTERED_USER_ID) 본인이다.**

이 명제는 다음 5 개 지점에 일관되게 반영되었다:

| 지점 | 이전 | 이후 |
|------|------|------|
| `CharterPropertyWriteService.updateProperty` | 누구나 수정 | 등록자만 수정 |
| `MonthlyPropertyWriteService.updateProperty` | 누구나 수정 | 등록자만 수정 |
| `CharterRecommendationService.canEdit` 산출 | 인증된 누구나 | 등록자 본인만 |
| `MonthlyRecommendationService.canEdit` 산출 | 인증된 누구나 | 등록자 본인만 |
| `property_board.js` 의 `canEdit` | 인증된 누구나 | 등록자 본인만 |

상태 변경(`changeStatus`) 은 이미 등록자 본인 검증을 수행하고 있었으므로 추가 변경 없음. 매물 비활성화 시 그 매물의 방문 예약 자원을 일괄 무효화하는 연동(`withdrawAllActiveWindowsForProperty`) 은 신규 호출.
