### 2.7 Scheduler (1 개) — `scheduler/` 하위

| 파일 | 동작 |
|------|------|
| `SlotExpirationScheduler.java` | `@Scheduled(fixedDelay = 60_000)` — 1 분 주기. 종료 시각 경과 슬롯을 식별해 (1) 슬롯 → CLOSED, (2) 슬롯에 묶인 CONFIRMED 예약 → COMPLETED (VISIT_RESULT 는 NULL 유지, 등록자 분류 대기), (3) 활성 구독 → EXPIRED (사유 SLOT_CLOSED) |

`@EnableScheduling` 은 기존 `WherehouseApplication` 에 이미 적용되어 있어 추가 작업 불필요.

## 6. 동시성 제어 — 1 차 범위 외, 그러나 백스톱 준비됨

설계 명세서 8.4 가 명시 위임한 동시성 제어 기법 선택·도입은 본 1 차 범위 외. 그러나 본 1 차에 다음이 사전 마련되어 있어, 추후 03_constraints.sql 적용 + 응용 동시성 제어 도입 시 코드 변경 최소화로 연결된다.

| 항목 | 위치 | 효과 |
|------|------|------|
| `UQ_VISIT_RESERVATION_CONFIRMED_SLOT` (부분 유일 인덱스) | `03_constraints.sql` | 슬롯당 CONFIRMED 예약 1 건을 DB 차원에서 강제 |
| `UQ_REOPEN_SUBSCRIPTION_ACTIVE` (2 컬럼 부분 유일) | `03_constraints.sql` | (슬롯, 탐색자) 활성 구독 1 건을 DB 차원에서 강제 |
| `UQ_VISIT_SLOT_WINDOW_START` | `03_constraints.sql` | 윈도우 내 같은 시작 시각 슬롯 중복 INSERT 차단 |
| `DataIntegrityViolationException` 핸들러 | `GlobalExceptionHandlerVisit.handleIntegrity` | DB 백스톱 발동 시 자동으로 409 + E7002 응답으로 변환 |

위 핸들러는 본 1 차에 이미 들어 있으므로, 본인이 03_constraints.sql 만 추가 적용하면 응용 코드 수정 없이 곧바로 백스톱이 켜진다.
